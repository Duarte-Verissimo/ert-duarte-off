# Glossary

**Intake**  
Processo inicial de submissão e recolha de informação sobre um pedido.

**Discovery**  
Fase de análise inicial para determinar se a informação submetida é suficiente.

**Authentication**  
Processo de verificação da identidade de um utilizador.

**Authorization**  
Verificação das permissões de um utilizador para executar uma ação.

**RBAC**  
Modelo de controlo de acessos em que as permissões dependem da role atribuída ao utilizador.

**Role**  
Conjunto de permissões atribuídas a um utilizador.

**Approval**  
Ação de aceitar ou validar um pedido submetido.

**Reopen**  
Ação de reabrir um pedido previamente fechado.

**Override**  
Ação excecional de sobrescrever dados ou decisões, sujeita a autorização, reautenticação e justificação.

**Audit Log**  
Registo de eventos relevantes, como tentativas não autorizadas, incluindo utilizador, ação e data/hora.

**Requirement**  
Descrição formal de uma necessidade que o sistema deve cumprir.

**Acceptance Criteria**  
Condições objetivas que definem quando um requisito é considerado cumprido.

**Test Case**  
Conjunto de passos para verificar um comportamento do sistema.

**BDD**  
Abordagem de testes baseada em comportamento, usando linguagem natural estruturada.
